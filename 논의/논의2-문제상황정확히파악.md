# 논의2: 문제 상황 정확히 파악

**날짜**: 2025-12-16

## 사용자가 보고한 문제

### 1️⃣ 번개 버튼 문제
**사용자 말씀**: "번개를 누르면 설정창으로 넘어가"

**제 이해**:
- 볼트를 바꾸고 싶어서 번개(⚡) 버튼을 누름
- 하지만 설정창만 열림
- 실제로 볼트가 바뀌지 않음

**질문**: 이게 맞나요? 아니면 다른 문제인가요?

---

### 2️⃣ 핫키 문제
**사용자 말씀**: "다른 파일을 누르고 볼트 카운슬 핫키를 누르면 동작을 안하는데 우리 플로그인을 종료하고 다시 핫키를 눌러야 그 파일로 넘어가"

**시나리오**:
```
1. Vault Council을 열어둔 상태
2. Note A를 보고 있음
3. Note B로 이동 (클릭)
4. Vault Council 핫키를 누름
5. ❌ Note B의 내용이 분석되지 않음 (여전히 Note A 내용)
6. Vault Council을 닫음
7. 다시 핫키를 누름
8. ✅ 이제 Note B가 분석됨
```

**제 이해**:
- Vault Council이 열려있을 때
- 다른 파일로 이동해도
- 이전 파일의 내용을 계속 보고 있음
- 플러그인을 껐다 켜야 새 파일 내용이 보임

**질문**: 이게 맞나요?

---

### 3️⃣ Custom Mode (여러 파일 선택)
**사용자 말씀**: "원래 여러파일을 선택해서 분석할수도 있기로 했었는데 그것은 구현이 되지 않았던것 같아"

**제 이해**:
현재 3가지 모드가 있어야 함:

**Auto Mode (현재 작동 중) ✅**
- 지금 보고 있는 파일
- + 그 파일에 링크된 파일들
- 자동으로 분석

**Folder Mode (현재 작동 중) ✅**
- 특정 폴더를 지정
- 그 폴더 안의 모든 마크다운 파일
- 자동으로 분석

**Custom Mode (미구현) ❌**
- 사용자가 직접 파일을 선택
- 예: Note1, Note5, Note10 이렇게 골라서
- 선택한 파일들만 분석

**질문**: 이게 맞나요? Custom Mode가 필요한가요?

---

## 정리

제가 이해한 게 맞는지 확인해주세요:

1. **번개 버튼**: 볼트 바꾸고 싶은데 설정만 열려서 불편함
2. **핫키 문제**: 다른 파일로 이동했는데 Vault Council이 업데이트 안됨
3. **Custom Mode**: 여러 파일 직접 선택하는 기능이 안 만들어짐

맞으면 바로 수정 시작하겠습니다!
